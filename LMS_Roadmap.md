## 🎯 هدف نسخه MVP

هدف این نسخه، رساندن سریع یک محصول قابل فروش برای دوره‌های ضبط‌شده است؛ بدون زیرساخت‌های پرهزینه کلاس آنلاین، WebRTC، ترنسکد خودکار و CI/CD پیچیده. تمرکز MVP روی موارد زیر است:

- فروش دوره‌های ویدیویی ضبط‌شده.
- احراز هویت، نقش‌ها و دسترسی دانشجو/مدرس/ادمین.
- آپلود دستی فایل‌های ویدیو در کیفیت‌های آماده.
- پخش ویدیو از URLهای ثابت و محافظت‌شده.
- پرداخت، ثبت‌نام در دوره و مشاهده محتوای خریداری‌شده.
- استقرار ساده روی PaaS یا Docker Compose.

## 🟡 فاز 2 — مدیریت دوره‌ها و محتوا

### 2.1 مدل دیتابیس دوره‌ها

- [x] جدول `courses`:
  ```text
  id, title, slug, description, teacher_id, price, discount_price,
  thumbnail, status, created_at
  ```
- [x] جدول `course_sections`:
  ```text
  id, course_id, title, order
  ```
- [x] جدول `lessons` به‌عنوان مدل جلسه/ویدیو با ساختار MVP:
  ```text
  id, section_id, title, order,
  content_type (video/pdf/text),
  quality_720_url  (required),
  quality_1080_url (optional),
  quality_480_url  (optional),
  duration_seconds,
  is_free_preview,
  allow_download,
  created_at
  ```
- [x] حذف نیاز به جداول پردازش ویدیو مانند `video_versions` و `video_processing_jobs` از MVP.
- [x] جدول `enrollments` برای ثبت دسترسی دانشجو به دوره.
- [x] جدول `course_reviews` برای نظرات و امتیازدهی.

### 2.2 APIهای دوره و محتوا

**عمومی:**

- [x] `GET /courses` — لیست دوره‌ها با صفحه‌بندی، فیلتر و مرتب‌سازی.
- [x] `GET /courses/:slug` — جزئیات دوره.
- [x] `GET /courses/:id/reviews` — نظرات دوره.

**مدرس:**

- [x] `POST /courses` — ایجاد دوره.
- [x] `PUT /courses/:id` — ویرایش دوره.
- [x] `POST /courses/:id/sections` — افزودن سرفصل.
- [x] `POST /sections/:id/lessons` — افزودن جلسه با URLهای کیفیت آماده.
- [x] `DELETE /lessons/:id` — حذف جلسه.
- [x] `PUT /lessons/:id` — ویرایش URLهای کیفیت، عنوان، ترتیب و تنظیمات دانلود.

**دانشجو:**

- [x] `GET /my/courses` — دوره‌های خریداری‌شده.
- [x] `GET /courses/:id/lessons/:lessonId` — دریافت محتوای مجاز پس از بررسی ثبت‌نام.
- [x] `POST /courses/:id/reviews` — ثبت نظر.

## 🟠 فاز 2.5 — مدیریت ساده رسانه و پخش ویدیو

### 2.5.1 تصمیم معماری MVP

در MVP هیچ پردازش سروری برای ویدیو انجام نمی‌شود. مدرس یا تیم محتوا، فایل ویدیو را خارج از سیستم آماده می‌کند و لینک هر کیفیت را به‌صورت دستی در پنل وارد می‌کند.

- [x] حذف FFmpeg، Worker و Queue از مسیر اصلی محصول.
- [x] حذف HLS اجباری از MVP.
- [x] استفاده از لینک مستقیم یا Signed URL برای فایل‌های MP4 آماده.
- [x] کیفیت 720p کیفیت پایه و اجباری است.
- [x] کیفیت‌های 480p و 1080p اختیاری هستند.

### 2.5.2 ساختار URLهای ویدیو

- [x] `quality_720_url`: لینک اجباری و کیفیت پیش‌فرض پلیر.
- [x] `quality_1080_url`: لینک اختیاری برای کاربران با اینترنت بهتر.
- [x] `quality_480_url`: لینک اختیاری برای اینترنت ضعیف‌تر.
- [x] اعتبارسنجی URLها در API ایجاد/ویرایش جلسه.
- [x] تصمیم MVP: پشتیبانی از URL مستقیم یا Signed URL آماده؛ بک‌اند فقط پس از بررسی دسترسی URLها را برمی‌گرداند.
- [x] در MVP، Signed URL توسط Storage/PaaS تولید یا در فیلد کیفیت ذخیره می‌شود و API فقط دسترسی کاربر را کنترل می‌کند.

### 2.5.3 پلیر ویدیو در فرانت‌اند

- [x] استفاده از پلیر سبک native HTML5 بدون وابستگی جدید برای MVP.
- [x] نمایش گزینه‌های کیفیت فقط برای URLهای موجود.
- [x] شروع پخش با `quality_720_url` به‌عنوان کیفیت پیش‌فرض.
- [x] تعویض کیفیت با تغییر `src` ویدیو بدون وابستگی به HLS.
- [x] نمایش پیام مناسب اگر فقط کیفیت 720p موجود باشد.
- [x] رعایت دسترسی: کاربر فقط پس از خرید دوره، مالکیت مدرس یا پیش‌نمایش رایگان به URL دسترسی می‌گیرد.

### 2.5.4 دانلود محتوا

- [x] اگر `allow_download=true` بود، نمایش دکمه دانلود برای کیفیت‌های موجود.
- [x] ثبت نام فایل و کیفیت در پاسخ API.
- [x] در MVP، لینک دانلود از همان URL مجاز/آماده استفاده می‌کند و قابلیت Signed URL در لایه Storage نگه داشته می‌شود.
